HTML和CSS学习笔记

HTML

一、互联网原理

1.上网（Internet-surfing）
其实是请求数据的过程，所谓的网页不过是一堆文件，通过HTTP协议在服务器和客户机间传递文件。
2.浏览器（browser）
（1）进行网页渲染、发起接受HTTP请求，有专门的本地文件夹接收从服务器传输的文件，
（2）五大浏览器：谷歌（chrome），火狐（Firefox），IE，苹果（Safari），欧朋（opera）
3.服务器（server）
本质是一台超大容量、超高性能的计算机，用作数据存储及运输。
4.HTTP协议
（1）超文本传输协议（HyperText Transfer Protocol）
（2）一次HTTP协议的实现过程包括请求（request）和响应（response）
*浏览一个网站往往同时发送多条HTTP请求。

二、HTML基础

1.纯文本
只保存文字内容不保存样式的文件。（.txt）一个中文占两个字节。
2.HTML
（1）超文本标记语言（HyperText Markup Language）
（2）描述文档语义的语言，在浏览器中不会显示，在编辑器中以普通文本显示。
（3）具体样式由CSS控制，HTML只是赋予文本语义。
3.HTML结构
（1）基本结构

tittle区域为浏览器标签页上的显示内容，body区域为具体页面的内容，这两个区域为用户可见区域。
（2）文档DTD
位置：HTML文件的第一行内容
标志：
名称：文档定义类型（DocType Definition）。
内容：定义了该文档使用的HTML规范的版本。
手册：www.w3school.cn（中文网站）
版本：HTML1.0~5.0，4.0版本时将结构与样式分离，常用HTML4.01。衍生出XHTML1.0在4.01版本上进行严格话与扩展升级。
在这两个常用规范下衍生出三个小规范，分别为Strict（严格版）、Transitional（过渡版）、Frameset（框架集版）
4.html标签

中文字符集包括：
UTF-8:国际通用字库，涵盖了所有人类的语言文字。一个汉字3个字节。
gb2312/gbk:中国字库，国家标准字库，涵盖了汉字和一些常用外文、符号。一个汉字2个字节。
*制作外文网站使用UTF-8，如果网站含有大量中文且要求加载速度，使用gbk。
（3）title标签：是一个双标签，显示网页的标题。同时tittle里的关键字与搜索引擎优化（SEO）有关。

（1）html是一个双标签，需要开始和结束
（2）xmlns:xml的namespace规定了标签使用的命名规范。用户浏览器不一致，必须规范什么是标签什么是内容。
（3）xml:lang:规定标签使用的语言。
5.head标签
（1）是一对双标签，对网页进行配置
（2）meta标签：是一个单标签，其中的charset定义了当前文件所使用的字符库。

三、HTML语法

1.标签之间对换行及空格不敏感，HTML只识别标签之间的嵌套。
2.空白折叠：普通文本之间的所有空格或换行将被压缩为一个空格。
3.标签分类：容器级（放置任意内容、标签）、文本级（只能放置文本、图片、表单元素）
4.标签属性：
（1）标签属性为键值对格式，k代表属性名，v代表属性值。
（2）键值对的格式是 k=”v”，尽量用双引号。
（3）一个标签可以有多个属性。
（4）一个属性可以有多个属性值，在双引号内部用空格间隔。
（5）属性书写在标签的起始标签内部，与标签名间用空格间隔，属性之间也需要空格间隔。

五、列表

列表形式：无序列表，有序列表，定义列表
1.无序列表
（1）<ul><li></li></ul>
（2）添加一个无序列表的语义。
（3）无序列表是一组标签，包括ul（unordered list）和li（list）两个成对出现的标签。
（4）li嵌套于ul标签。
（5）li标签属于容器级标签，甚至还能嵌套ul、li。
2.有序列表
（1）<ol><li></li></ol>
（2）添加一个有序列表的语义。
（3）有序列表是一组标签，包括ol（ordered list）和li（list）
（4）li嵌套于ol标签。
（5）li标签属于容器级标签，甚至还能嵌套ol、li。
3.定义列表
（1）有三个标签dl、dt、dd组成。
（2）<dl>
<dt>主题a</dt>
<dd>说明a</dd>
<dt>主题b</dt>
<dd>说明b</dd>
</dl>
（3）添加一个定义列表的结构，里面有定义主题，对主题的解释。
（4）dl：definition list，定义列表。
dt：definition title，定义标题。
dd：definition description，定义说明，解释最近的dt。
（5）这组标签需要同时出现，dl里嵌套了dt和dd。
（6）dt后可以不跟dd，也可跟多个dd。
4.div和span标签
div：
（1）div是大的范围。div：division，没有特殊语义，用来布局。
（2）div里经常放置一些具有特殊功能的标签，这就是布局过程。
（3）div是容器级标签。
span：
（1）span是小范围跨度，用于小范围布局。
（2）span是文本级标签。一般认为p>span>a

六、表格

1.表格基础
（1）table、tr、td，三者嵌套关系table>tr>td
（2）tr：table rows，行，td：table dock，单元格。
（3）th标签：表示表头单元格，添加一个表头语义，与td同等地位。
2.合并单元格
（1）单元格标签：td、th。
（2）有两个属性用来合并单元格：
rowspan：合并行单元格，属性值是数字，表示合并几行个单元格。
colspan：合并列单元格，属性值是数字，表示合并几个列单元格。
上边线一致的单元格属于同一行。

七、表单

表单是网页上提供用户输入、选择的控件。
1.form
（1）所有表单元素都需要写在form内部，是一个功能型标签而不是结构型标签，
（2）功能：规定我们提交的数据发送到哪里，发送的方法是哪种。
method：提交方法，属性值为get和post。
action：提交的位置。
2.文本框
（1）允许用户输入文本。
（2）input标签：输入，有一个属性type，属性值不同代表不同的表单元素。

<input type="text" value="默认文本" />
3.密码框
（1）input标签的type属性值为password。
4.按钮
（1）按钮有三种，普通、提交、重置
普通type：button
提交type：submit
重置type：reset
（2）value值为显示在按钮上的提示文本。
5.单选框
（1）同一组单选框只能有一个选择。
（2）type：radio
（3）同一组单选框必须以name属性加以区分。
6.多选框
（1）同一组多选框可以有多个选择。
（2）type：checkbox
（3）同一组多选框必须以name属性加以区分。
7.文本域
（1）标签textarea，可以输入多行文本。
（2）rows：属性值是数字，表示有多少行。
（3）cols：属性是数字，表示每行可以有多少字节。一个汉字两字节。
8.下拉菜单
（1）是一组标签，必须同时出现，有嵌套关系。
（2）select：选择，option：选择，只能select中嵌套option。两者都是双标签。

CSS

一、css基础知识

1.css基础概念
（1）css：cascading style sheets，层叠样式表。
（2）作用：规定了标签在网页上的显示样式。
（3）前端三层：
html：结构层，打造网页的整体架构。
css：样式层，装饰页面。
JavaScrip：行为层，一些页面交互效果。
2.css的作用
（1）重要概念：层叠式，样式（html标签在页面的显示效果）。
（2）将需要的样式的属性值直接罗列出来。
（3）样式设置的两个关键，选择器和样式表。
3.css的几个小样式
（1）文本样式：
大小：font-size。本身是一个复合属性font，里面的单一属性需要用font单一属性名。
颜色：color。属性值有多种选择方案，十六进制，RGB码，rgba，颜色的英文名。
字体：font-family。复合属性的一个单一属性，中文字体和英文字体。书写时英文字体在前，中文字体在后。
（2）盒子实体化基本属性：
宽度：width。属性值是像素为单位。
高度：height。属性值是像素为单位。
背景色：background-color。属性值是颜色值。
边框：border。复合属性，有多个属性值，为宽度，颜色。
4.css的书写位置
（1）根据书写css属性位置的不同，将我们的样式表分为三种：行内式、内嵌式、外链式。
行内式：css所有属性书写在一个标签的一个叫style属性里，不同属性间用；分割。
内嵌式：写在html文件里的style标签中，所有的css属性都写在<style>css样式</style>这对双标签内部，用{}括起。此时style标签有一个属性type，值为text/css。
外链式：将css单独写在一个.css文件里，再用link标签引入。
5.css注意事项
（1）css的键值对：k:v;
（2）多个属性值间必须用;（分号）分割。
（3）内嵌式所有的样式必须写在一对大括号内。
（4）代码上传时为了速度需要去除空格空行，进行代码压缩。

二、基础选择器

1.标签选择器：指我们选择需要添加样式的标签元素模式。
（1）用法：标签名{}
（2）选中的是页面所有的对应标签，不管嵌套方式多复杂。
（3）用途：进行初始样式的设置，默认样式的清除。
2.id选择器：通过标签的id属性值来选择相应的标签。
（1）书写：#开头，后面紧跟id名，中间没有任何空格。#para{}
（2）页面内标签id唯一，所以id选择器只能选择一个标签。
（3）id属性名需要遵循一个原则，必须以字母开头，严格区分大小写，可以是数字下划线或者横线。
3.类选择器：通过标签的class属性来选择标签。
（1）写法：.开头，后面跟class属性值，不能有空格。
（2）class命名规则与id相同，一个class可以有多个值，多个类名之间用空格隔开。
（3）类上样式，id上行为。
4.通配符*：可以选中包括body在内的所有标签
（1）简单案例里清空默认样式。

三、高级选择器

包括后代选择器、交集选择器、并集选择器
1.后代选择器
（1）后代选择器又叫包含选择器。
（2）通过标签之间的嵌套、层级关系缩小选择范围。
（3）书写：祖先 后代{} 中间有一个空格。
（4）空格左侧是右侧的祖先元素，并非一定要父子关系。
（5）一般祖先用类名，后代用标签。
（6）可以多层选择。
2.并集选择器
（1）既满足条件1又满足条件2。
（2）两种或以上选择器同时存在于一个标签上。
（3）写法：选择器之间直接连接，没有任何符号。
（4）所选择的基础选择器可以是标签和类的连续或类与类的连续。
（5）兼容问题：IE6不兼容类的连续交集。
3.并集选择器
（1）满足条件1或者满足条件2。
（2）很多元素可能又相同的css样式，通过并集选择器统一设置。
（3）书写：不同选择器之间用,连接。

四、继承性和层叠性

1.继承性
（1）祖先元素拥有的样式后代元素也会继承。
（2）后代元素能继承来自祖先元素的文字样式，不能继承盒子的样式。
（3）继承性是层叠性的第一个特性。
2.层叠性
（1）多个选择器选中一个元素，相同的元素该依据谁的样式？
（2）选择器权重：权重高的层叠掉权重低的。
id选择器>类选择器>标签选择器>通配符
（3）对于有高级选择器的情况
①选择器都选中了元素
首先比较权重（数基础选择器的个数，依次比较其权重）先比较id的个数，再比较类的个数，最后比较标签的个数。如果有并集，则以实际选中的元素为准。
如果权重相同，则看书写顺序，书写在后面的层叠掉前面的。
②如果没选中元素，样式靠继承
第一种：祖先元素距离目标距离不同，继承最近的祖先元素。
第二种：距离一样比较权重。
第三种：距离权重都一样，则比较书写顺序，后写的层叠先写的。
③关键字!important
给单一属性添加，会给这个属性权重加到最大。
